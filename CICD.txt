STEP 1: Prepare local
·         $ mkdir my-project && cd my-project
·         $ git init
·         (Add files: web_wrapper.py, asa_vpn_ops.py, Dockerfile, requirements.txt, .gitignore)
·         $ git add .
·         $ git commit -m "Initial commit"
·         $ git branch -M main
STEP 2: Create GitHub repo
·         GitHub → New repository → my-project
·         Copy HTTPS URL
STEP 3: Connect and push
·         $ git remote add origin <GitHub-URL>
·         $ git push -u origin main
STEP 4: Create Azure VM
·         Azure Portal → Virtual Machines → Create
·         (Choose: Ubuntu 22.04, B1s, SSH key = id_rsa.pub)
STEP 5: Get VM IP
·         Azure Portal → VM → Public IP address
·         Note this IP (e.g., 13.245.123.456)
STEP 6: SSH to VM and install Docker
·         $ ssh -i id_rsa ubuntu@13.245.123.456
·         $ sudo apt install docker.io -y
·         $ sudo usermod -aG docker ubuntu
·         $ logout
STEP 7: Prepare app directory on VM
·         $ ssh -i id_rsa ubuntu@13.245.123.456
·         $ mkdir /home/ubuntu/app
·         $ cd /home/ubuntu/app
·         $ git init
·         $ git remote add origin https://github.com/user/my-project.git
·         $ exit
STEP 8: Add GitHub secrets
·         GitHub → Settings → Secrets → Actions
·         Add:
·           - VM_HOST = 13.245.123.456
·           - VM_USER = ubuntu
·           - VM_SSH_KEY = (content of id_rsa)
STEP 9: Create .github/workflows/deploy.yml
·         (Use content from section 11)
·         $ git add .github/
·         $ git commit -m "Add workflow"
·         $ git push origin main
STEP 10: Add Azure NSG rules
·         Azure Portal → VM → Networking
·         Add 2 inbound rules (SSH port 22, HTTP port 80)
STEP 11: Test deployment
·         Make small code change:
·         $ git add .
·         $ git commit -m "Test"
·         $ git push origin main

·         Watch: GitHub → Actions tab
·         Should see:
·           ✅ Build job success
·           ✅ Deploy job success
STEP 12: Access app
·         Open browser: http://13.245.123.456
·         Should see your Flask app!


13. COMMON COMMANDS
GIT:
·         $ git status (see changes)
·         $ git log (see commits)
·         $ git push origin main (upload)
·         $ git pull origin main (download)
DOCKER:
·         $ docker build -t myapp . (create image)
·         $ docker run -d -p 80:5000 myapp (start)
·         $ docker ps (list running)
·         $ docker logs myapp (see output)
·         $ docker stop myapp (stop)
SSH:
·         $ ssh -i id_rsa ubuntu@IP (login to VM)
·         $ scp -i id_rsa file ubuntu@IP:/path/ (copy file)
AZURE CLI:
·         $ az vm list (list VMs)
·         $ az vm start -n name -g rg (start VM)


14. TROUBLESHOOTING
Problem: "Permission denied (publickey)"
Check SSH secret has full key (BEGIN/END lines)
Problem: "Docker command not found"
sudo usermod -aG docker ubuntu, then logout/login
Problem: "Connection refused"
Check Azure NSG rule for port 80 is added
Problem: "App not accessible"
Check container running: docker ps, Check logs: docker logs myapp
Problem: "Build fails"
Check requirements.txt has all packages


15. SECURITY CHECKLIST
·         ✅ Never commit .env files
·         ✅ Never commit private SSH key
·         ✅ Use GitHub secrets for credentials
·         ✅ Restrict NSG port 22 to your IP (not 0.0.0.0)
·         ✅ Use strong SSH keys (4096-bit)
·         ✅ Rotate secrets annually
·         ✅ Keep system updated: apt update && apt upgrade




































===============================
PROJECT DEPLOYMENT PLAN
===============================

Goal:
Deploy automation application on Azure VM and make it accessible
via URL with secure device connectivity.


Step 1: Infrastructure Setup
- Create Azure Linux VM (Ubuntu)
- Open ports: 22 (SSH), 80 (HTTP), 443 (HTTPS)


Step 2: Application Deployment
- Clone project from GitHub to VM
- Setup backend (Python service)
- Deploy frontend using Nginx


Step 3: Application Flow
- User accesses application via browser (URL)
- Frontend loads from Nginx
- Frontend calls backend APIs
- Backend executes automation tasks


Step 4: Device Connectivity
- Backend connects to firewall devices (ASA / Palo Alto)
- Connection happens via VPN or internal network


Step 5: Update Process
- Developer pushes code to GitHub
- VM pulls latest code
- Backend is restarted
- Frontend is updated


Step 6: Security
- Only Nginx (frontend) exposed to internet
- Backend runs internally
- Device communication secured via VPN


Final Summary:
Azure VM hosts frontend (Nginx) and backend (Python),
users access via URL, backend securely connects to network devices,
updates handled via Git workflow.

git clone https://github.com/manish-epicor/Automation.git

cd backend

===============================

STEP 7: FRONTEND DEPLOY
===============================
cd ../frontend
sudo cp -r * /var/www/html/


===============================
STEP 8: CONFIGURE NGINX
===============================
sudo nano /etc/nginx/sites-available/default

Paste:

server {
    listen 80;
    root /var/www/html;
    index login.html;

    location / {
        try_files $uri $uri/ =404;
    }

    location /api/ {
        proxy_pass http://localhost:5000/;
    }
}

===============================
sudo systemctl restart nginx


===============================

===============================
http://<public-ip>/login.html
