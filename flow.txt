- Runs ASA show commands
- Discovers existing VPN tunnels

- Prints tunnel summary table

↓
menu.py

- Returns selected option

↓
Workflow Modules (Based on Menu Selection)

1) reset.py
   - Clears IKEv1 and IPsec SAs
   - Verifies tunnel status post reset

2) update.py
   - Captures PRE-change state
   - Optional PSK update
   - Optional ACL update
   - Captures POST-change state
   - Prints change summary

3) build.py
   - Takes VPN parameters
   - Builds encryption ACL
   - Configures tunnel-group
   - Saves ASA configuration

4) troubleshoot.py
   - Displays IKE status
   - Displays IPsec status
   - Shows crypto-related logs

5) Exit
   - Breaks menu loop

↓
disconnect_asa()
- Gracefully closes ASA SSH connection
- Ends CLI automation

## A) CLI FLOW (Direct Terminal Run → ASA → Menu Loop)

```
User runs CLI (local machine / jump server)
↓
run_asa.py
  - Switches to project root directory (BASE_DIR)
  - Executes module: backend.projects.vpn_automation.asa.main
↓
main.py
  - Prints CLI banner
  - connect_asa()  -> client.py
  - run_asa_vpn_ops(conn) -> asa_vpn_ops.py
  - finally: disconnect_asa(conn) -> client.py
↓
asa_vpn_ops.py
  - discover_tunnels(conn) -> discovery.py  (first)
  - while True:
      show_menu() -> menu.py
      user selects:
        1 → reset_vpn(conn, tunnels)       -> reset.py
        2 → update_vpn(conn)               -> update.py
        3 → build_vpn(conn)                -> build.py
        4 → troubleshoot_vpn(conn)         -> troubleshoot.py
        5 → Exit (break)
↓
disconnect_asa() -> client.py
END (CLI flow)
```

### Benefit of CLI Flow
- Best for developers / network engineers
- Full guided menu, interactive inputs
- Fast troubleshooting and operations without browser dependencies

VPN AUTOMATION PLATFORM – ARCHITECTURE
=================================================

-This project a uses simple layered architecture, where each layer has a
clear responsibility and communicates only with the adjacent layer.
-It uses a FastAPI backend along with a wrapper layer
 to reuse existing Python CLI automation and provide a web-based VPN automation solution.
-------------------------------------------------
1. FRONTEND LAYER:- Provides a simple web UI for users
-------------------------------------------------
Technology:
- HTML → Defines the structure of the web pages
- CSS  → Styles the UI and controls layout and appearance
- JavaScript → Handles user actions, navigation, and sends API requests to the backend

Purpose:
- Provides a simple web UI for users
- Users can select:
  - Project
  - Firewall (ASA / Palo Alto)
  - VPN action (Reset / Build / Update / Troubleshoot)

Function:
- Collects user input
- Sends HTTP requests to backend using fetch()
- Displays output and logs to the user

-------------------------------------------------
2. BACKEND API LAYER
-------------------------------------------------
Technology:
- FastAPI (Python)
- File: backend/main.py

Purpose:
- Acts as the main backend entry point
- Exposes REST APIs like:
  - /login
  - /projects
  - /firewalls
  - /asa/{task}

Function:
- Receives HTTP requests from UI
- Validates input data (task name, optional parameters)
- Uses {task} to identify which VPN action to run
- Forwards requests to the wrapper layer

Note:
- This layer does NOT contain firewall CLI or API logic

-------------------------------------------------
3. WRAPPER LAYER
-------------------------------------------------
Technology:
- Python
- File: backend/projects/vpn_automation/asa/web_wrapper.py

Purpose:
- Connects FastAPI backend with existing automation scripts
- Acts as a bridge between web requests and CLI automation

Function:
- Receives task name from backend
- Calls the correct automation workflow (reset, build, update, etc.)
- Manages firewall connection (connect / disconnect)
- Captures full CLI output and returns it to backend

Key Benefit:
- Existing CLI code is reused without rewriting

-------------------------------------------------
4. AUTOMATION LOGIC LAYER
-------------------------------------------------
Technology:
- Python (.py files)

Location:
- backend/projects/vpn_automation/asa/

Purpose:
- Contains actual VPN automation logic

Functions:
- Discover VPN tunnels
- Reset VPN tunnels
- Build new VPN tunnels
- Update VPN configuration
- Troubleshoot VPN issues

-------------------------------------------------
5. DEVICE COMMUNICATION LAYER
-------------------------------------------------
Technology:
- Netmiko (Python library)

Purpose:
- Provides SSH-based CLI communication with Cisco ASA firewalls
- Sends commands and receives output securely

Vendor Support:
- Cisco ASA → CLI over SSH (Netmiko)
- Palo Alto → API-based support (planned)

-------------------------------------------------
6. FIREWALL / INFRASTRUCTURE LAYER
-------------------------------------------------
Devices:
- Cisco ASA firewall (current)
- Palo Alto firewall (future)

Function:
- Executes actual VPN operations on the device
- Applies configuration and operational commands

-------------------------------------------------
END-TO-END FLOW (SIMPLE)
-------------------------------------------------

User clicks button in UI
↓
Frontend sends HTTP request (fetch)
↓
FastAPI backend receives and validates request
↓
{task} identifies VPN action (reset / build / update / troubleshoot)
↓
web_wrapper.py triggers automation scripts
↓
Python automation scripts run CLI logic
↓
Netmiko sends commands over SSH
↓
Firewall executes VPN operation
↓
CLI output returned to backend
↓
Backend sends result to UI
↓
User sees output and logs

-------------------------------------------------
KEY DESIGN POINTS
-------------------------------------------------
- Clean separation of layers
- Reuse of existing CLI automation
- Simple and modular design
- Easy to extend for new firewall vendors
- Suitable for UI-driven and API-driven automation

Summary:-
------------------------
- Physical interfaces carry encrypted VPN traffic over the Internet (underlay).
- Logical interfaces represent the VPN tunnel where traffic is decrypted (overlay).
- ASA uses policy-based VPN (crypto-map).
- Palo Alto uses route-based VPN (tunnel interface).

  def _show(conn, cmd: str) -> str:
      last_exc = None
      for attempt in (1, 2):
          try:
              out = conn.send_command(cmd)
              return out if out is not None else ""
          except Exception as e:
              last_exc = e
              if attempt == 1:
                  time.sleep(0.35)
      return f"__ERROR__:{last_exc}"
